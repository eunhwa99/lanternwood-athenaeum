# 2026-05-27 Full Port Plan

## Scope

Port the Lanternwood Athenaeum dashboard to the requested event contract, runtime controls, Codex follow-up context, guarded coordinator permissions, compact inspector/details UX, stable layout, scene speech bubbles, and matching unit/e2e coverage.

## Non-goals

- Do not add live OpenAI credentials, secrets, or new API-key-backed tooling.
- Do not add dependencies or broad framework changes.
- Do not replace the Pixi scene or redesign unrelated agent personas beyond the requested event/UI/runtime behavior.

- Project docs and setup: `.gitignore`, `.env.example`, `README.md`, `docs/codex-cli-setup.md`, `package.json`, `server/waitForCodexHealth.ts`.

- Tests and snapshots: relevant unit/integration tests plus `tests/e2e/lanternwood.spec.ts` and `tests/e2e/__snapshots__/lanternwood-dashboard.png`.

## Constraints

- Follow `AGENTS.md` and `.agents/docs/harness-engineering.md`.
- Keep the main agent as orchestrator; use bounded read/review workers for non-atomic areas.
- No new dependencies.
- Preserve existing patterns and avoid unrelated refactors.
- Implement and verify by phase rather than touching everything at once.

## Worker Personas

- Runtime explorer: event contract, RunAdapter, task id, abort, previous-run, and Codex SSE paths.
- Server explorer: global agents, coordinator policy, prompt injection, and backend validation paths.
- UI/scene explorer: inspector, drawer, layout, Pixi debug hooks, bubbles, and e2e surfaces.
- Main orchestrator: integrates patches, owns tests, verification, and review routing.

## Phase 1: Runtime Foundation

1. Add stable short task id helper and tests for long prompts/collision-like inputs.
2. Update `RunAdapter.startRun(input, options)` with `signal` and `previousRun`.
3. Add mock adapter abort behavior and `agent.prompted` sequence coverage.
4. Forward abort signal and previous run context through Codex adapter/backend body.
5. Validate Codex SSE events before yielding, failing malformed events with `Invalid AgentEvent from Codex SSE`.
6. Preserve reported specialist output and fail unfinished active specialists on cancellation/failure.

## Phase 2: Global Agents And Coordinator

1. Add optional `server/globalAgents.ts` loader for `LANTERNWOOD_AGENTS_HOME` or `~/.agents`.
2. Add deterministic `server/coordinatorPolicy.ts` defaults and inference for safe Obsidian note/project create/delete intents.
3. Inject global persona/policy and previous-run context into specialist and synthesis prompts.
4. Emit `permission.reviewed` events and render them in the inspector.

## Phase 3: UI Detail Surfaces

1. Add shared `src/ui/runDetails.ts` for report, prompt, diagnostics, raw extraction, redaction, truncation, and run log entries.
2. Replace inline raw details in compact cards with buttons that open `RunDetailDrawer`.
3. Add drawer tabs for final output, agent reports, prompts, raw Codex, and run log.
4. Keep final output preview compact and make full output copyable in the drawer.
5. Stabilize responsive layout so scene, input, inspector, and final output do not overlap.

## Phase 4: Scene And E2E

1. Make mock and Codex workflows emit `agent.prompted` before specialist work/report routes.
2. Show prompt/report speech bubbles from payload excerpts for 3-4 seconds.
3. Sort agent sprites by y-depth and expose `zIndex` in debug state.
4. Expose `__LANTERNWOOD_DEBUG_BUBBLES__`.
5. Update Playwright assertions and snapshot for new layout, missing visible timeline, bubbles, Luma dispatch movement, depth sorting, and drawer content.

## Phase 5: UI Refinement Pass

1. Remove the standalone Final Output panel from the dashboard grid.
2. Show `Open full final output` as a compact action inside the library scene when Luma has produced final output.
3. Keep inspector cards, drawer sections, and long text surfaces fixed-size with internal scrolling instead of dynamic growth.
4. Restore agent color chips in the top agents summary using the registry colors.
5. Queue dispatch prompt speech bubbles so Luma carries each bubble while moving toward the target agent, then advances sequentially.
6. Resize the in-flight stop control so it sits beside `Send to Luma` with matching compact button geometry.
7. Refresh unit/e2e coverage and the screenshot snapshot for the revised layout.

## Phase 6: Selective Routing Pass

1. Add a typed `route.planned` event with selected/skipped agents, rationale, and confidence.
2. Add deterministic Luma routing rules for Orion, Neria, Quill, and Argus with low-confidence fallback.
3. Emit `route.planned` before any `agent.prompted` events in mock and Codex workflows.
4. Prompt only selected agents; leave skipped agents idle instead of marking them done.
5. Let synthesis consume partial selected reports and explicitly note skipped agents.
6. Add a Routing tab and run-log entries so users can inspect why agents were selected or skipped.
7. Update unit/e2e coverage and snapshot for selective routing and selected-agent-only scene movement.
